Why the gaps exist

  • Random → PLAID/TACHIOM gap (≈10×): Centroid ANN concentrates the candidate set near the query direction. A random 1% sample retrieves ~1% of oracle docs; centroid probing retrieves the top ~83% with the same fraction because it selects the most aligned docs.
  • PLAID → WARP gap: Centroid centroids represent the average of a cluster of token embeddings. Oracle-relevant docs have tokens that are specifically aligned with query tokens — their exact similarity is high, but their assigned centroid may be displaced (averaged away from the peak). WARP's Xtr threshold tests exact pairwise similarities, avoiding this approximation error. In practice: WARP needs O(N) to compute the threshold scan; the speedup comes entirely from reducing the expensive MaxSim reranking set.
  • TACHIOM's budget-allocation advantage (real IR scenarios): In natural language, vocabulary follows a power law: a small fraction of word types ("tail" types) are highly discriminative but rare. TACHIOM allocates centroid budget ∝ 1/frequency — rare types get more centroids per doc. For a query whose key discriminative tokens are rare, TACHIOM finds relevant docs with fewer total candidates than PLAID. This advantage grows with vocabulary imbalance and is less visible in the uniform synthetic data used here.
  • Implication for system design: If latency budget allows ~5–20% scanning, PLAID/TACHIOM deliver near-oracle quality. If you need <1% candidates (the high-speed regime), WARP's exact threshold provides a fundamentally better quality-efficiency frontier — at the cost of an O(N) threshold scan rather than O(K) centroid lookup.
