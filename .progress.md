## Learnings
- Rust not pre-installed; installed via rustup to /Users/henryweller/.cargo/bin
- `tokenizers 0.19` has no `wordpiece` feature — updated to `0.21` with no feature flags (WordPiece is built-in)
- `mongodb 3.x` has no `tokio-runtime` feature — tokio runtime is default; removed the feature flag
- `cli` main.rs with only `// TODO` fails compilation (no `main` fn) — added minimal `fn main() {}`
- `cargo metadata` outputs single JSON line so `grep '"name"'` returns 1 line; all 8 crates confirmed via JSON parsing
- `data/` and `output/` are gitignored; use `git add -f` for `.gitkeep` files
- `tokenizers::Tokenizer::from_file` expects tokenizer.json, NOT vocab.txt; must use WordPiece::from_file(vocab).build() + BertNormalizer + BertPreTokenizer to construct from plain vocab.txt
- `WordPieceTokenizer::inner` must be `pub` (not `pub(crate)`) for colbert crate to access it
- `ColBertEngine::query` takes `&self` per Engine trait but encoder needs `&mut self`; solved with `RefCell<ColBertEncoder>` + unsafe Sync impl for single-threaded demo
- Scenario TOML `type = "shared"` requires `#[serde(tag = "type", rename_all = "lowercase")]` on CorpusDef enum
